## 📋 完整測試流程（15 分鐘）

### 第1步：註冊與登入

1. **開啟瀏覽器**
   ```
   http://localhost:5173
   ```
   應該自動導向登入頁面

2. **註冊新帳號**
   - 點擊「註冊」連結
   - 填寫：
     - 使用者名稱：`測試使用者`
     - Email：`test123@example.com`（使用新的 email）
     - 密碼：`password123`
     - 確認密碼：`password123`
   - 點擊「註冊」
   - ✅ **預期結果**：自動登入並進入主頁面

3. **檢查持久化登入**
   - 重新整理頁面（F5 或 Cmd+R）
   - ✅ **預期結果**：仍然保持登入狀態（不會被導回登入頁）

---

### 第2步：咖啡廳 CRUD

4. **查看咖啡廳列表**
   - 主頁應該顯示咖啡廳列表
   - ✅ **預期結果**：看到之前測試時建立的咖啡廳

5. **新增咖啡廳**
   - 點擊「+ 新增咖啡廳」
   - 填寫：
     - 名稱：`前端測試咖啡廳`
     - 類別：`辦公友善`
     - 描述：`這是從前端新增的咖啡廳`
     - 地址：`台北市大安區羅斯福路三段`
     - 評分：`4.5`
     - 價格等級：`$$`
     - 勾選：WiFi、插座、光線充足、經常有座位
   - 點擊「新增」
   - ✅ **預期結果**：
     - 回到主頁
     - 看到新增的咖啡廳
     - 重新整理頁面，咖啡廳還在（資料已存到後端）

6. **編輯咖啡廳**
   - 點擊剛新增的咖啡廳
   - 點擊「✏️ 編輯」
   - 修改名稱為：`前端測試咖啡廳（已修改）`
   - 點擊「更新」
   - ✅ **預期結果**：
     - 回到主頁
     - 名稱已更新
     - 重新整理頁面，更新仍保存

7. **查看咖啡廳詳情**
   - 點擊任一咖啡廳
   - ✅ **預期結果**：看到完整資訊，包含所有設施標籤

---

### 第3步：到訪記錄

8. **記錄到訪**
   - 在咖啡廳詳情頁
   - 點擊「📝 記錄到訪」
   - 填寫：
     - 日期：今天的日期
     - 評分：5 星
     - 備註：`環境很棒，咖啡也好喝！`
   - 點擊「送出記錄」
   - ✅ **預期結果**：顯示「已記錄到訪！」

9. **查看到訪記錄**
   - 點擊頂部「📝 我的到訪」
   - ✅ **預期結果**：
     - 看到剛才的記錄
     - 顯示咖啡廳名稱、地址、評分、備註
     - 統計資訊正確

10. **刪除到訪記錄**
    - 點擊某筆記錄的 🗑️ 按鈕
    - 確認刪除
    - ✅ **預期結果**：記錄消失

## 🐛 常見問題

### Q1: 前端顯示「未提供認證 Token」？

**A:** 可能原因：
1. 沒有登入
2. Token 已過期
3. localStorage 被清除

**解決方法**：重新登入

---

### Q2: 無法新增咖啡廳？

**A:** 檢查：
1. 是否已登入
2. 後端是否正在運行
3. CORS 設定是否正確
4. 瀏覽器 Console 是否有錯誤

---

### Q3: 重新整理後資料消失？

**A:** 檢查：
1. 後端資料庫檔案 `dev.db` 是否存在
2. 後端是否正常運行
3. API 請求是否成功

---

### Q4: 看不到其他使用者的到訪記錄？

**A:** 這是**正常的**！基於隱私和權限：
- 咖啡廳：公開（所有人都能看到）
- 到訪記錄：私人（只能看到自己的）
- 願望清單：私人（只能看到自己的）

## 🔧 除錯技巧

### 查看 API 請求

1. 打開瀏覽器開發者工具（F12）
2. 切換到 "Network" 標籤
3. 執行操作（如新增咖啡廳）
4. 查看請求：
   - 請求 URL
   - 請求 Headers（有沒有 Token）
   - 回應狀態碼
   - 回應內容

### 查看 localStorage

在瀏覽器 Console 執行：
```javascript
// 查看 Token
localStorage.getItem('token')

// 查看使用者資訊
localStorage.getItem('user')

// 清除認證（測試用）
localStorage.clear()
```

### 查看後端資料庫

```bash
cd /Users/hannah/wp1141/hw4/backend

# 使用 SQLite CLI 查看資料
sqlite3 dev.db "SELECT * FROM users;"
sqlite3 dev.db "SELECT * FROM cafes;"
sqlite3 dev.db "SELECT * FROM visits;"
sqlite3 dev.db "SELECT * FROM wishlist;"
```
