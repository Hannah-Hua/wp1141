# Edit Profile 功能說明

## ✅ 已實作功能

- ✅ **姓名** - 可修改（從 OAuth 取得）
- ✅ **個人簡介（Bio）** - 最多 160 字元，即時顯示字數

## 🔧 技術實作

### 圖片處理流程

1. **選擇圖片**
   - 用戶點擊上傳按鈕
   - 觸發檔案選擇器（隱藏的 `<input type="file">`）
   
2. **圖片預覽**
   - 使用 `FileReader` API 讀取檔案
   - 轉換為 Data URL（base64）
   - 立即顯示在介面上

3. **提交資料**
   - 將圖片轉換為 base64 字串
   - 連同姓名和 Bio 一起送到 API
   - API 儲存到 MongoDB

### 資料格式

```typescript
{
  name: string,           // 姓名
  bio: string,            // 個人簡介（最多 160 字元）
  image: string,          // 大頭貼 base64 或 URL
  coverImage: string      // 背景圖 base64 或 URL
}
```

### API 端點

```
PATCH /api/users/[userId]
```

**請求範例：**
```json
{
  "name": "ziyenhua b12705022",
  "bio": "這是我的個人簡介",
  "image": "data:image/jpeg;base64,/9j/4AAQSkZJRg...",
  "coverImage": "data:image/jpeg;base64,/9j/4AAQSkZJRg..."
}
```

## 📦 儲存方案

### 目前實作：Base64 儲存
- 圖片以 base64 字串形式儲存在 MongoDB
- **優點**：簡單、不需額外服務
- **缺點**：資料庫會變大、查詢較慢

### 未來改進建議：雲端儲存
考慮整合以下服務：
- **Cloudinary** - 免費額度、自動優化
- **AWS S3** - 可靠、可擴展
- **Vercel Blob** - 與 Vercel 整合

**改進後的流程：**
1. 上傳圖片到雲端儲存
2. 取得圖片 URL
3. 只儲存 URL 到資料庫

## ⚠️ 注意事項

### 檔案大小限制
- **背景圖**：最大 5MB
- **大頭貼**：最大 2MB
- 超過限制會顯示警告

### 圖片品質
- Base64 儲存會增加資料庫大小
- 建議圖片壓縮後再上傳
- 未來可加入自動壓縮功能

### 瀏覽器支援
- 使用 `FileReader` API（現代瀏覽器都支援）
- IE 11 以下不支援
