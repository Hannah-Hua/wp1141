# 效能優化說明

## 🔧 已實作的優化

### 1. 批量查詢優化（解決 N+1 問題）

**問題：**
- 原本每個貼文都單獨查詢用戶資訊（50 個貼文 = 50 次查詢）
- 每個貼文都單獨計算轉發次數（50 個貼文 = 50 次查詢）
- 總共可能高達 100+ 次資料庫查詢

**優化後：**
- 收集所有需要的用戶 ID，一次性批量查詢（1 次查詢）
- 使用聚合查詢批量計算所有轉發次數（1 次查詢）
- 總共只需要 3 次資料庫查詢（貼文列表 + 用戶批量查詢 + 轉發次數聚合）

**效能提升：**
- 從 O(n) 次查詢降為 O(1) 次查詢
- 預期響應時間從 2-8 秒降為 < 500ms

### 2. 資料庫索引優化

**新增索引：**
```javascript
// Post Model
PostSchema.index({ repostBy: 1, createdAt: -1 });  // 優化 repost 查詢
PostSchema.index({ originalPost: 1 });              // 優化轉發次數計算
PostSchema.index({ author: 1, parentPost: 1, repostBy: 1 }); // 複合索引優化 Following 查詢
```

**現有索引：**
- `{ author: 1, createdAt: -1 }` - 用戶貼文查詢
- `{ createdAt: -1 }` - 時間排序
- `{ parentPost: 1 }` - 回覆查詢

## 🔍 優化細節

### 批量查詢用戶資訊

**優化前：**
```javascript
// 每個貼文都查詢一次（N+1 問題）
posts.map(async (post) => {
  const author = await User.findById(post.author);  // 50 次查詢
  const reposter = await User.findById(post.repostBy);  // 額外查詢
});
```

**優化後：**
```javascript
// 收集所有用戶 ID，一次性查詢
const userIds = new Set();
posts.forEach(post => {
  userIds.add(post.author);
  if (post.repostBy) userIds.add(post.repostBy);
});

const users = await User.find({ _id: { $in: Array.from(userIds) } });  // 1 次查詢
const userMap = new Map();  // 建立映射表供快速查找
```

### 批量計算轉發次數

**優化前：**
```javascript
// 每個貼文都計算一次（N+1 問題）
posts.map(async (post) => {
  const repostCount = await Post.countDocuments({ originalPost: post._id });  // 50 次查詢
});
```

**優化後：**
```javascript
// 使用聚合查詢一次性計算所有轉發次數
const repostCounts = await Post.aggregate([
  { $match: { originalPost: { $in: postIds } } },
  { $group: { _id: '$originalPost', count: { $sum: 1 } } }
]);  // 1 次查詢
```

## ⚠️ 注意事項

### 索引建立
- 索引會在第一次使用時自動建立
- 如果資料量很大，建立索引可能需要一些時間
- 索引會佔用額外的儲存空間，但大幅提升查詢速度

### MongoDB 連接
- 確保 MongoDB 連接池設定正確
- 使用 MongoDB Atlas 時，注意連線數限制
