# 改進說明

## 1. 降級訊息出現的原因

### 可能的原因

「劇情之神似乎暫時離開了，請稍後再試。」這個降級訊息會在以下情況出現：

1. **API 配額用盡**
   - 錯誤訊息：`LLM_QUOTA_EXCEEDED` 或 `GEMINI_QUOTA_EXCEEDED`
   - 用戶看到：`⚠️ 劇情之神暫時無法回應（API 配額已用完），請稍後再試。`

2. **API 回應超時**
   - 錯誤訊息：`LLM_TIMEOUT` 或 `GEMINI_TIMEOUT`
   - 用戶看到：`⏱️ 劇情之神回應超時，請稍後再試。`

3. **JSON 解析錯誤**
   - 錯誤訊息：`GEMINI_JSON_PARSE_ERROR` 或 `OPENAI_JSON_PARSE_ERROR`
   - 原因：LLM 的回應不是有效的 JSON 格式
   - 用戶看到：`⚠️ 劇情之神的回應格式有誤，請稍後再試。`

4. **回應格式不符合預期**
   - 錯誤訊息：`GEMINI_INVALID_FORMAT` 或 `OPENAI_INVALID_FORMAT`
   - 原因：LLM 的回應缺少必要欄位（narration、options、effects）
   - 用戶看到：`⚠️ 劇情之神的回應格式不符合預期，請稍後再試。`

5. **其他未知錯誤**
   - 任何其他未預期的錯誤
   - 用戶看到：降級回應訊息

### 如何除錯

所有錯誤都會記錄在伺服器日誌中，包括：
- 錯誤類型
- 錯誤訊息
- 錯誤堆疊
- LLM 原始回應（如果是 JSON 解析錯誤）

查看日誌可以幫助了解具體的錯誤原因。

### 改進措施

1. ✅ 增加了詳細的錯誤日誌記錄
2. ✅ 區分不同類型的錯誤，提供更明確的錯誤訊息
3. ✅ 記錄 LLM 原始回應，方便除錯

## 2. 文字被截斷的問題

### 問題說明

LINE 按鈕模板的文字限制：
- **按鈕模板文字**：最多 160 字元
- **文字訊息**：最多 5000 字元

之前的實作會直接截斷超過 160 字元的敘事，導致玩家看不到完整內容。

### 解決方案

已實作智能文字分割功能：

1. **長文字分割**
   - 如果敘事超過 160 字元，會先發送完整的敘事文字
   - 文字會智能分割成多個訊息（每個最多 4000 字元）
   - 分割時會優先選擇句號、換行符或空格作為分割點

2. **按鈕分離**
   - 敘事發送完畢後，再單獨發送按鈕選項
   - 確保玩家可以看到完整的劇情內容

### 運作流程

```
如果敘事 <= 160 字元：
  → 直接使用按鈕模板（敘事 + 按鈕）

如果敘事 > 160 字元：
  → 分割敘事為多個文字訊息（每個最多 4000 字元）
  → 發送所有敘事部分
  → 最後發送按鈕選項
```

### 範例

**之前（會被截斷）：**
```
[按鈕模板]
你走進椰林大道，發現天空中漂浮著一行行 SQL 語句...（只顯示前 160 字元）
[選項按鈕]
```

**現在（完整顯示）：**
```
[文字訊息 1]
你走進椰林大道，發現天空中漂浮著一行行 SQL 語句。這些語句像是有生命一般，在空中交織成複雜的資料結構...

[文字訊息 2]
你感受到一股強大的資料流從地底湧出，整個校園似乎都在運轉著某種巨大的資訊系統...

[按鈕模板]
請選擇你的行動：
[選項按鈕]
```

## 測試建議

1. **測試長文字**：
   - 讓 LLM 生成超過 160 字元的敘事
   - 確認文字完整顯示，沒有被截斷

2. **測試錯誤處理**：
   - 查看伺服器日誌，確認錯誤訊息清楚
   - 確認用戶收到適當的錯誤提示

3. **測試分割功能**：
   - 測試超過 4000 字元的超長敘事
   - 確認分割點選擇合理（不會在句子中間分割）
