# Chatbot 對話 / 功能設計

本文件說明 LINE Bot《遺失在台大地下室的秘寶》的對話邏輯、功能設計與訊息格式，作為助教與開發者的功能參考稿。

---

## 1. Bot 角色定位
- **身分**：平行台大的 Dungeon Master（DM），語氣友善、敘事細膩。
- **目標**：引導玩家完成主線、推進劇情，並在 LLM 失效時給出明確降級訊息。
- **語調**：第二人稱敘事（「你走進…」「你發現…」），帶有校園 × 地城冒險風。

---

## 2. 關鍵指令
| 使用者指令 | Bot 行為 |
|-----------|---------|
| `開始冒險` | 重設/初始化會話，送出開場劇情與命名指示。 |
| `重新開始` | 與 `開始冒險` 相同，重新建立會話資料。 |
| `1/2/3`    | 在「選擇職業」階段用來指定職業。 |
| 其他文本   | 視階段決定：若在遊戲中，交由 LLM 產生劇情；不然提示使用者輸入正確指令。 |

---

## 3. 對話流程（狀態機）
1. **idle**：等待 `開始冒險`。回覆歡迎訊息與指示。
2. **creating_name**：任何文字視為玩家名稱，儲存後進職業選擇。
3. **choosing_class**：僅接受 `1/2/3`，使用按鈕模板呈現三職業。
4. **playing**：所有輸入交給 LLM（`generateGameTurn`）→ 產生敘事/選項/effects → 套用遊戲效果。
5. **game_over**：提示遊戲結束，提醒輸入 `開始冒險` 重啟。
6. **錯誤/降級**：若 LLM 超時/配額/格式錯誤，回純文字，內容包含「劇情之神暫時離開」與提示。

---

## 4. 訊息型態
| 階段 | 型態 | 範例 / 說明 |
|------|------|-------------|
| 開場、錯誤 | 純文字 (`replyTextMessage`) | 固定敘事、降級提示等。 |
| 命名完成 → 選職 | Button Template | `replyClassSelectionMessage()`：引導回覆 1/2/3。 |
| 遊戲進行 | 多段文字 + Button Template | `replyGameOptionsMessage()`：總敘事分段、附餘 2-4 選項。降級時不附按鈕。 |
| 遊戲結束 | 純文字 | `replyGameOverMessage()`。 |

---

## 5. LLM 呼叫與回應格式
- 模型：`gpt-4o-mini`
- 內容：系統 prompt + 最近 2 則對話 + 玩家輸入。
- 回應 JSON（narration / options[] / effects）
  - `effects.progress_change` 正值推進進度；一旦 >=100 會進結局。
  - `effects.game_over` true 表示立即結束。
- Timeout：10 秒，若超時則觸發 `LLM_TIMEOUT` 流程。

---

## 6. 錯誤處理（劇情之神暫時離開）
| 錯誤代碼 | 行為 |
|----------|------|
| `LLM_TIMEOUT`、`LLM_QUOTA_EXCEEDED` | 回覆純文字（無選項），內容含「劇情之神…」。 |
| `OPENAI_JSON_PARSE_ERROR`、`OPENAI_INVALID_FORMAT` | 純文字提示格式錯誤，請稍後再試。 |
| 其他未知錯誤 | 使用 `getFallbackResponse()`，但因選項為空，前端只收到敘事文字。 |

---

## 7. 管理後台功能
- 路徑 `/admin`：以 Next.js Page 實作。
- API：
  - `GET /api/admin/conversations`（支援 lineUserId、日期、階段篩選）
  - `GET /api/admin/conversations/[id]`（單筆詳情）
  - `GET /api/admin/stats`（總使用者、會話數等）
- 前端呈現：
  - 對話清單 + 分頁
  - 點擊查看完整訊息（含 LLM 敘事、進度、HP、道具）
  - 基本統計卡片

## 10. 常見場景
- 玩家亂輸入：依階段提示（如要求輸入 1/2/3 或「開始冒險」）。
- 玩家在錯誤階段輸入「重新開始」：force reset → 回初始狀態。
- LLM 超時：純文字告知「劇情之神...請稍後再試」。
